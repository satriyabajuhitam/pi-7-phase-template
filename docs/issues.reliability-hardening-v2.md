## Dependency rules
- Upstream decomposition and PRD self-review guardrails should land before downstream `/issues` gate hardening so both sides of the handoff use the same approval language.
- `/issues` gate hardening should land before repo-wide docs cleanup so operator guidance reflects the final planning posture rather than an intermediate state.
- The `finish` helper should land after the planning/handoff tickets so its closeout vocabulary matches the newly hardened upstream workflow.
- Repo-level docs and guidance should wait until the underlying skill/prompt behavior is settled.
- Final workflow validation should wait until planning, closeout, and docs alignment are complete.
- Human judgment is required before declaring the v2 slice “ready for template use” across future repos.

- `Depends on`: `none` or explicit ticket IDs
- `Blocks`: explicit ticket IDs or blank when none
- `Parallelizable`: `yes`, `no`
- QA follow-up: reopen an existing ticket if the original scope still fits; otherwise add a new ticket
